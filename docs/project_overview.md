# Visão do Projeto

## Nome
AccessManager

## Objetivo

Sistema interno para controle de clientes e acessos (telas).

---

## Problemas resolvidos

- controle manual desorganizado
- dificuldade de identificar vencimentos
- dificuldade de gerenciar múltiplas telas por cliente
- falta de controle financeiro simples

---

## Solução

Sistema centralizado com:

- controle de telas
- controle técnico
- controle financeiro separado
- histórico de alterações

---

## MVP (Fase 1)

Inclui:

- clientes
- servidores
- telas
- renovações técnicas
- financeiro manual
- dashboard simples

---

## Fora do MVP

- automação com servidores
- integração com APIs externas
- notificações
- cobrança automática
